# 安全なPR作成コマンド

main/masterブランチから直接コミットしないように保護しつつ、PRを作成するよ！💪

## 手順

### 1. ブランチ確認
現在のブランチが `main` または `master` かチェックして！

```bash
git branch --show-current
```

### 2. 分岐処理

**main/masterの場合:**
1. 変更内容を確認して適切なブランチ名を提案
2. ユーザーに確認してから新しいブランチを作成
   ```bash
   git checkout -b <branch-name>
   ```

**それ以外の場合:**
- そのまま次のステップへ進む

### 3. コミット作成
- `git status`と`git diff`で変更を確認
- コミットメッセージはConventional Commits形式で提案
- ユーザー確認後にコミット

### 4. プッシュ
```bash
git push -u origin <branch-name>
```

### 5. PR作成
`gh`コマンドでPRを作成:
```bash
gh pr create --fill
```

または内容を確認してから:
```bash
gh pr create --title "タイトル" --body "説明"
```

## PR本文（簡潔に）
リポジトリにPR規約があればそれを優先。なければ次の軸だけで書く。設計メモ・経緯の長文は不要。レビュアーが「何が変わり・どこに効き・何が残るか」を数十秒で掴める長さに。
- 概要: 1〜2行（何を / なぜ）
- 変更点: 箇条書き（必要なら 本体 / 副作用 でグループ分け）
- 影響する処理・画面: 表（処理→影響）。「通常時は変化なし」も明記
- 残作業: 別チケット番号付き
- 検証・関連: 末尾に最小限

原則: 既存本文に**追記し続けて膨らませない**。情報が増えたら統合して書き直す。

## 注意事項
- 各ステップでユーザーに確認を取ること！🙋‍♀️
- ブランチ名はkebab-caseで、変更内容を表す名前にする
- Conventional Commits: `feat:`, `fix:`, `docs:`, `refactor:`, `test:`, `chore:`
